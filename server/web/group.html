    Uptime: device runtime, the duration from start testing till testing stopped. Testing stops when all tests are blocked to be continued without system manual reset.
    MTBF = Total running hours/Total failures
    Total failures= (critical issues) + (Non-Critical issues).
    Critial Issues: Phone reset including power down/cycle, kernel restart, system restart, etc; Phone lockup including UI freeze, black screen and application lockup.
    Non-Critical Issues: Application/process restart including force close, core dump (native process crash).
    Notes:
    1. In AT&T, no distinguish between critical and non-critical issues, MTBF testing stop once device meet failure.
    2. If strictly follow AT&T MTBF calculation, MTBF fail as no device runtime >50 hrs
